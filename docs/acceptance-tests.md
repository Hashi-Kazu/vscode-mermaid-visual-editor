## 3. 共通（ダイアグラム横断）受入テスト

| テストID | シナリオ | 期待結果 |
|---------|---------|----------|
| AT-C01 | フローチャートで右クリック→「SVGとして保存」を選択する | 保存ダイアログが開き、選択先にSVGファイルが書き出される（R-C06-01〜03） |

| AT-C03 | パネルで `.md` を開いた後、別エディタ／外部（共有ドライブ・Git pull）で同じファイルを変更し、その後パネル上で編集操作する（R-C09-02/03） | コンフリクト警告モーダルが表示され、「上書き」選択時のみ全文置換される。「最新を読み込む」選択またはダイアログを閉じた場合は自分の編集が中止され、ディスクの最新内容でパネルが再同期される |
| AT-C04 | AT-C03 でいずれかの解決を選ぶ（R-C09-04） | 破棄される側の内容が `<ファイル名>.conflict-<mine\|remote>-<timestamp>.md` として同じフォルダに退避される |
| AT-C05 | パネル操作中（ドラッグ等）に外部でファイルが変更され、操作完了時に書き込みが走る（R-C09-05） | 操作中の外部変更イベントは無視されるが、書き込み直前のbase比較でコンフリクトが検知され分岐する |
| AT-C06 | CRLF 改行のファイルと LF 改行で内容が同一の状態を作る（R-C09-02） | 改行コードのみの差異はコンフリクトとして扱われず、通常どおり書き込まれる（`npm test` の `conflictDetection.test.ts` で自動検証） |
