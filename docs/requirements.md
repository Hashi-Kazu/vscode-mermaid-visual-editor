## 1. 目的・スコープ

本文書は、Visual Studio Code向けMermaidエディタ拡張機能（`vscode-mermaid-visual-editor`）の要求仕様を定義する。

本拡張機能は、Markdownファイル（`.md`）内のMermaidコードブロックを視覚的に編集し、ソースコードと双方向に同期する手段を提供する。

### 実装フェーズ

| フェーズ | 内容 |
|---------|------|
| **フェーズ1（実装中）** | ガントチャートの双方向編集・ビューア操作（US-G01〜US-G14） |
| フェーズ2以降（後続） | ダイアグラム横断共通機能（US-C01〜US-C08）および他ダイアグラム型ビューア |

### ID 命名規則

本文書では以下のプレフィックスを使用する。ビューアが増えた際は新しいプレフィックスを追加し、各ビューアの仕様は独立した節として追記する。

| プレフィックス | 対象 | 例 |
|--------------|------|----|
| `US-C**` | ダイアグラム横断の共通ユーザーストーリー | US-C01 |
| `US-G**` | ガントチャート固有のユーザーストーリー | US-G01 |
| `R-G**`  | ガントチャート固有の要件 | R-G01-01 |
| `NF-**`  | 共通非機能要件 | NF-01 |
| `NF-G**` | ガントチャート固有の非機能要件 | NF-G01 |
| `AT-G**` | ガントチャートの受入テスト | AT-G01 |

## 5. ガントチャート機能要件

### 5.1 用語定義（ガントチャート固有）

| 用語 | 定義 |
|------|------|
| ガントパネル | ガントチャートをインタラクティブに操作するVS Code WebViewパネル |
| タスクバー | ガントチャート上で1タスクを示す横棒 |
| セクション | 複数タスクをグループ化する論理単位（`section` キーワード） |
| タスク | ガントチャート内の1行（ラベル・状態・ID・開始日・期間から構成） |

### 5.2 US-G01 ガントチャート表示

**ユーザーストーリー**  
ユーザーは、Mermaidのgantコードを開いたとき、インタラクティブなガントチャートを横並びで確認したい。なぜなら、日程の全体像を視覚的に把握しながら編集したいから。

| ID | 要件 | 優先度 | 受入条件 |
|----|------|--------|---------|
| R-G01-01 | Markdown内のgantブロックに対してガントパネルを開けること | 必須 | コマンド `mermaid.openGantt` またはエディタタイトルバーボタンからパネルが開くこと |
| R-G01-02 | ガントパネルはエディタの横（`ViewColumn.Beside`）に表示される | 必須 | 既に開いている場合は再利用すること |
| R-G01-03 | タスクを横棒（タスクバー）として時系列に描画する | 必須 | 開始日・期間に応じて正しい位置・幅で表示されること |
| R-G01-04 | セクション単位でタスクをグループ化して表示する | 必須 | セクション名が行ラベルとして表示されること |
| R-G01-05 | 日付軸（タイムライン）を上部に表示する | 必須 | 年月・週単位の目盛りが表示されること。表示範囲はタスク全体を包含すること |
| R-G01-06 | タスクの状態（`done` / `active` / `crit` / 未指定）を色で区別する | 必須 | 各状態に対して視覚的に異なるスタイルが適用されること |
| R-G01-07 | VS Codeのカラーテーマ（ダーク/ライト）に追従する | 推奨 | VS Code標準のCSS変数を使用していること |
| R-G01-08 | Mermaidコードの変更から 500ms 以内にパネルが更新される | 必須 | デバウンス処理を実装し、連続入力中は更新を遅延させること |
| R-G01-09 | ガントパネル上の操作完了時に `.md` ファイルを自動保存する | 必須 | タスク移動・リサイズ・名称変更・追加・削除のいずれの操作後も、VS Codeのダーティマーカーが残らないこと。セクション並び替え・Undo等の構造編集（structuralEdit）後も同様に保存されること |
| R-G01-10 | タスクバーのラベルが省略（…）表示されている場合、マウスオーバーでツールチップに全文を表示する | 必須 | バー幅に収まっているラベルにはツールチップを表示しないこと |
| R-G01-11 | ガントブロックが存在しないファイルをパネルで開いた場合、「ガントチャートを挿入」ボタンを表示する | 必須 | ボタンクリックで本日起点のデフォルトテンプレート（タイトル・2タスク）がファイルに挿入され、即座にガントパネルに反映されること。挿入と同時にファイルが自動保存されること |

### 5.3 US-G02 タスクバーのドラッグ移動（日程変更）

**ユーザーストーリー**  
ユーザーは、タスクバーを左右にドラッグして開始日を変更したい。なぜなら、コードを直接編集せず直感的にスケジュールを調整したいから。

| ID | 要件 | 優先度 | 受入条件 |
|----|------|--------|---------|
| R-G02-01 | タスクバーを左右にドラッグして開始日を変更できる | 必須 | ドラッグ中にバーがリアルタイムで移動すること |
| R-G02-02 | ドラッグ中は変更後の開始日をツールチップで表示する | 必須 | `dateFormat` に従った形式で日付が表示されること |
| R-G02-03 | ドラッグ完了時にMermaidコードの開始日が更新される | 必須 | タスクが `after <id>` 形式の相対指定の場合は絶対日付に変換して書き込むこと |
| R-G02-04 | スナップ機能：ドラッグ時に最小単位（1日）でスナップする | 必須 | 半日未満のドラッグは前日、半日以上は翌日にスナップすること |
| R-G02-05 | `Shift` キーを押しながらドラッグすると週単位でスナップする | 推奨 | 7日単位で吸い付く動作となること |

---
### 5.4 US-G03 タスクバーのリサイズ（期間変更）

**ユーザーストーリー**  
ユーザーは、タスクバーの右端をドラッグして期間（終了日）を変更したい。なぜなら、作業量の調整をビジュアルで行いたいから。

| ID | 要件 | 優先度 | 受入条件 |
|----|------|--------|---------|
| R-G03-01 | タスクバー右端のリサイズハンドルをドラッグして期間を変更できる | 必須 | リサイズハンドルはバー右端6pxの領域とし、カーソルが `ew-resize` に変わること |
| R-G03-02 | 最小期間は 1日 とする | 必須 | 1日未満に縮小しようとした場合は 1日 に固定すること |
| R-G03-03 | リサイズ中は変更後の終了日をツールチップで表示する | 必須 | `dateFormat` に従った形式で日付が表示されること |
| R-G03-04 | リサイズ完了時にMermaidコードの期間（`nd` / `nw` 等）が更新される | 必須 | 日数で計算し `Xd` 形式で書き込むこと |

### 5.5 US-G04 タスクのインライン編集・状態変更

**ユーザーストーリー**  
ユーザーは、ガントパネル上でタスク名の編集や状態の切り替えを直接行いたい。なぜなら、ビューを切り替えずにタスクの内容・進捗を修正したいから。

| ID | 要件 | 優先度 | 受入条件 |
|----|------|--------|---------|
| R-G04-01 | タスクバー（状態インジケーター以外の領域）またはラベル列のタスク名をダブルクリックしてタスク名のインライン編集ができる | 必須 | ダブルクリックでテキスト入力フィールドが表示されること。状態インジケーターのクリックとは独立して動作すること。バー上とラベル列のどちらからでも編集開始できること |
| R-G04-02 | 編集内容がMermaidコードのタスクラベルに即時反映される | 必須 | Enterキー押下、Escapeキー以外のキーによるフォーカスアウト、またはバー以外の領域をクリックしたときにMermaidコードが更新されること。バー外へのクリックはmousedownキャプチャにより確実に確定されること |
| R-G04-03 | Escapeキーで編集をキャンセルできる | 必須 | Escape押下で変更を破棄し元のテキストに戻ること |
| R-G04-04 | 空文字列への変更は無効とする | 必須 | 空文字でEnterを押した場合は変更を適用しないこと |
| R-G04-05 | タスクバーの状態インジケーター（バー左端のカラー部分）をクリックすると状態選択ピッカーが表示される | 必須 | タスク名のダブルクリック編集とは独立した操作として動作すること |
| R-G04-06 | 状態ピッカーでは `done` / `active` / `crit` / 未設定（なし）の4択を選択できる | 必須 | 現在の状態にチェックまたはハイライトが付くこと |
| R-G04-07 | 状態を選択するとMermaidコードの状態キーワードが更新され、タスクバーの色が即時変わる | 必須 | 未設定を選んだ場合はコードから状態キーワードが除去されること |

### 5.6 US-G05 タスクの追加・削除

**ユーザーストーリー**  
ユーザーは、ガントパネル上でタスクを追加・削除したい。なぜなら、ビューを切り替えずにスケジュールを拡張・整理したいから。

| ID | 要件 | 優先度 | 受入条件 |
|----|------|--------|---------|
| R-G05-01 | セクション行または任意の行で右クリック→「タスクを追加」でタスクを追加できる | 必須 | 追加タスクのデフォルトは「新しいタスク」・期間7日・開始日は直前タスクの翌日 |
| R-G05-02 | 追加直後にタスク名のインライン編集が自動的に開始される | 必須 | R-G04 のインライン編集が即座に起動すること |
| R-G05-03 | ツールバーの「+ タスク追加」ボタンからもタスクを追加できる | 推奨 | 追加先はアクティブなセクションの末尾とすること |
| R-G05-04 | 右クリック→「タスクを削除」またはコンテキストメニュー表示中・タスクバー上で `Delete` キーを押してタスクを削除できる | 必須 | 削除後にMermaidコードから対象タスク行が除去されること |
| R-G05-05 | ルートレベル（セクションなし）にもタスクを追加できる | 必須 | セクションが存在しない場合でも追加操作が動作すること |
| R-G05-06 | コンテキストメニューはビューポートの端で折り返し表示する | 必須 | メニューが画面下端・右端に近い場合、表示位置を上または左にシフトして画面内に収まること |

### 5.7 US-G06 セクション管理

**ユーザーストーリー**  
ユーザーは、ガントパネル上でセクションを追加・名称変更したい。なぜなら、フェーズや担当者でタスクを整理したいから。

| ID | 要件 | 優先度 | 受入条件 |
|----|------|--------|---------|
| R-G06-01 | ツールバーまたは右クリックメニューから「セクションを追加」できる | 必須 | Mermaidコードに `section セクション名` 行が追加されること |
| R-G06-02 | セクション名をダブルクリックでインライン編集できる | 必須 | 編集確定後にMermaidコードの `section` 行が更新されること |
| R-G06-03 | セクションラベルのクリックまたは折りたたみアイコンで配下タスクを折りたたみ・展開できる | 必須 | 折りたたむと配下のタスク行が非表示になること。折りたたみ状態はビュー上の一時状態であり、Mermaidコードは変更しないこと。パネル再描画（コード変更・Undo等）により折りたたみ状態はリセットされること |
| R-G06-04 | セクションラベル列のドラッグハンドルを上下にドラッグしてセクション順序を変更できる | 必須 | 配下のタスクも一緒に移動すること。ドラッグ中はドロップ位置を水平ラインで示すこと |
| R-G06-05 | セクションを右クリック→「セクションを削除」またはセクションラベル上で `Delete` キーを押してセクションを削除できる | 必須 | 配下のタスクも合わせて削除され、Mermaidコードから `section` 行およびタスク行が除去されること |

### 5.8 US-G07 Undo / 双方向同期の競合管理

**ユーザーストーリー**  
ユーザーは、ガントパネル上の操作を `Ctrl+Z` で取り消したい。また、テキストエディタとガントパネルを同時に編集しても壊れないことを期待する。

| ID | 要件 | 優先度 | 受入条件 |
|----|------|--------|---------|
| R-G07-01 | Webview内に最大50操作のUndoスタックを保持する | 必須 | 51操作目以降は最古のエントリーが破棄されること |
| R-G07-02 | 各操作（移動・リサイズ・名称変更・追加・削除）の直前にスナップショットをUndoスタックに積む | 必須 | 1回の `Ctrl+Z` で1操作分だけ戻ること |
| R-G07-03 | `Ctrl+Z` でMermaidコードをスナップショット時点の内容に復元する | 必須 | Undo後にテキストエディタの内容もスナップショット時点の状態になること |
| R-G07-04 | ガントパネルの操作中（`isOperating = true`）はMermaidコード変更イベントを無視する | 必須 | ドラッグ・リサイズ中にコード変更イベントが割り込んでもパネルが中断されないこと |
| R-G07-05 | 操作完了後、MermaidコードをソースオブトゥルースとしてパネルをReparse・再描画する | 必須 | 操作完了後、コードと同期した最新の状態がガントパネルに反映されること |
| R-G07-06 | テキストエディタ側で直接コードを変更した場合、500ms後にパネルが最新状態に更新される | 必須 | デバウンス後にパースしてパネルに反映されること |

### 5.9 US-G08 マイルストーン

**ユーザーストーリー**  
ユーザーは、ガントチャートにマイルストーン（期限・リリース日など）を設定したい。なぜなら、重要な節目をタスクと区別して視覚的に管理したいから。

| ID | 要件 | 優先度 | 受入条件 |
|----|------|--------|---------|
| R-G08-01 | 右クリックメニュー「マイルストーンを追加」でマイルストーンを追加できる | 必須 | Mermaidコードに `milestone` キーワード付きのタスク行が追加されること |
| R-G08-02 | マイルストーンをダイヤモンド形（◆）またはひし形アイコンでタスクバーと区別して表示する | 必須 | タスクバー（横棒）と視覚的に明確に異なること |
| R-G08-03 | マイルストーンは期間を持たず、日付1点として描画される | 必須 | バー幅を持たず、指定日の位置に縦軸アイコンとして表示されること |
| R-G08-04 | マイルストーンのラベルをダブルクリックでインライン編集できる | 必須 | R-G04 と同等の編集動作とすること |
| R-G08-05 | マイルストーンの日付をドラッグで変更できる | 推奨 | 左右ドラッグで日付が変わりMermaidコードに反映されること |

### 5.10 US-G09 タスクの並び替え

**ユーザーストーリー**  
ユーザーは、同一セクション内のタスクを上下に並び替えたい。なぜなら、優先順位や工程順を視覚的に整理したいから。

| ID | 要件 | 優先度 | 受入条件 |
|----|------|--------|---------|
| R-G09-01 | タスクラベル列のドラッグハンドルアイコンを上下にドラッグして並び順を変更できる | 必須 | ドラッグハンドルはつかむカーソル（grab）で表示し、ドラッグ中はドロップ位置を水平ラインで示すこと |
| R-G09-02 | 並び替え完了時にMermaidコードのタスク行順・セクション所属が更新される | 必須 | ドロップ後、コードの行順とセクション帰属がパネルの表示と一致すること |
| R-G09-03 | セクションをまたいだ並び替えができる | 必須 | 別セクションへドロップした場合、そのセクションの指定位置に移動しMermaidコードも更新されること |
| R-G09-04 | 右クリックメニュー「↑ 上へ移動」「↓ 下へ移動」で1行ずつ移動できる | 推奨 | 先頭タスクの「上へ」・末尾タスクの「下へ」はグレーアウトされること |

### 5.11 US-G10 ビューのパン・ズーム

**ユーザーストーリー**  
ユーザーは、ガントパネル内でビューを移動・拡大縮小したい。なぜなら、長期スケジュールや大量タスクでも全体を把握しながら作業したいから。

| ID | 要件 | 優先度 | 受入条件 |
|----|------|--------|---------|
| R-G10-01 | タイムライン上（タスクバー以外の領域）を左クリックドラッグでビューを水平・垂直に移動できる | 必須 | タスクバーのドラッグ（日程変更）と競合しないこと。タスクバー以外の領域でのみパンが有効となること |
| R-G10-02 | `Ctrl` キーを押しながらマウスホイールを操作すると拡大・縮小できる | 必須 | カーソル位置を中心にズームすること。ピクセル/日の比率が変化し、タスクバーの幅と位置が再計算されること |
| R-G10-06 | `Ctrl` キーなしのマウスホイールは上下スクロールとして動作する | 必須 | ブラウザ標準のスクロール動作（縦方向）を妨げないこと |
| R-G10-03 | ズームの最小倍率・最大倍率を設ける | 必須 | 最小: 7px/日（月単位が見渡せる）、最大: 60px/日（日単位が見やすい）とすること |
| R-G10-04 | ツールバーの「全体表示」ボタンでガントチャート全体が画面に収まるよう自動調整する | 推奨 | 全タスクが収まる倍率・スクロール位置に自動調整されること |
| R-G10-05 | ツールバーの「リセット」ボタンでビューの表示位置・拡大率を初期状態に戻せる | 必須 | 初期状態はデフォルト倍率（デフォルトpx/日）、スクロール位置は「今日 − 3日」が左端に見えること。パネル起動時・コード更新時も同じ位置に自動スクロールすること |

### 5.12 US-G11 タスク依存関係

**ユーザーストーリー**  
ユーザーは、タスク間の依存関係（先行タスクが終わったら後続タスクが始まる）をガントパネル上で設定・確認したい。なぜなら、工程の前後関係を視覚的に把握・管理したいから。

| ID | 要件 | 優先度 | 受入条件 |
|----|------|--------|---------|
| R-G11-01 | `after <taskId>` 形式の依存関係を矢印でタイムライン上に可視化する | 必須 | 依存元タスクの終端から依存先タスクの始端へ矢印が描画されること。依存元が複数のタスクにまたがる場合もそれぞれ矢印を表示すること |
| R-G11-02 | タスクの右クリックメニュー「依存関係を設定」から依存元タスクを選択できる | 必須 | 選択UIに全タスクのラベル（またはID）が一覧表示されること。設定後にMermaidコードのタスク行が `after <id>` 形式の開始日に更新されること |
| R-G11-03 | タスクの右クリックメニュー「依存関係を削除」で依存関係を解除し絶対日付に変換できる | 必須 | 解除後にMermaidコードが `after <id>` から解決済みの絶対日付形式に更新されること |
| R-G11-04 | 依存関係を持つタスクは `after <id>` を解決した絶対日付でタイムライン上に描画する | 必須 | パーサーが `after <id>` を参照先タスクの終了日に変換し、正しい位置にバーが表示されること |
| R-G11-05 | 依存関係によって算出された開始日を持つタスクをドラッグ移動すると、依存関係が解除されて絶対日付に変換される | 必須 | ドラッグ完了後にMermaidコードが `after <id>` から絶対日付形式に変わること |

### 5.13 US-G12 ガントヘッダー設定の編集

**ユーザーストーリー**  
ユーザーは、ガントパネル上でタイトルや軸書式を変更したい。なぜなら、ソースを直接編集せずに基本設定をビューアから完結させたいから。

| ID | 要件 | 優先度 | 受入条件 |
|----|------|--------|---------|
| R-G12-01 | ガントパネル左上のコーナーセル（タイトル表示領域）をダブルクリックしてタイトルをインライン編集できる | 必須 | 編集確定後にMermaidコードの `title` 行が更新されること。タイトルを空文字に変更した場合は `title` 行が除去されること |
| R-G12-02 | ツールバーのドロップダウンで `axisFormat` を選択できる（デフォルト・年/月・月/日・年-月-日・月-日のプリセット） | 必須 | 選択変更時にMermaidコードの `axisFormat` 行が更新されること。「デフォルト」選択時は `axisFormat` 行が除去されること |
| R-G12-03 | ヘッダー設定の変更はUndoスタックに積まれ `Ctrl+Z` で取り消せる | 必須 | 1回のUndoで変更前の設定に戻ること |

### 5.14 US-G13 タスクの複製

**ユーザーストーリー**  
ユーザーは、既存タスクをコピーして1行下に追加したい。なぜなら、同じ期間・状態の類似タスクを反復作成する手間を省きたいから。

| ID | 要件 | 優先度 | 受入条件 |
|----|------|--------|---------|
| R-G13-01 | タスクバーまたはラベル上で右クリック→「⊕ タスクを複製」でタスクを直下に複製できる | 必須 | 複製タスクは同セクション内の元タスクの直後に追加されること |
| R-G13-02 | 複製タスクの開始日は元タスクの終了日（翌日）とし、期間・状態は元タスクと同一とする | 必須 | IDは空文字（自動付番）で複製されること。依存関係（`afterId`）は複製されないこと |
| R-G13-03 | 複製直後にタスク名のインライン編集が自動起動する | 必須 | R-G04-01 と同等の編集動作が即座に起動すること |

---
### 5.15 US-G14 日付・期間の直接入力

**ユーザーストーリー**  
ユーザーは、タスクの開始日と期間を数値で直接指定したい。なぜなら、ドラッグ操作では長期スケジュールで日単位の精度が出せないから。

| ID | 要件 | 優先度 | 受入条件 |
|----|------|--------|---------|
| R-G14-01 | タスクの右クリックメニュー「✎ 日程を編集」でポップアップを開き、開始日（`type="date"` ネイティブ入力）を直接指定できる | 必須 | ポップアップは画面端で折り返し表示されること。ポップアップ外クリックでキャンセルされること |
| R-G14-02 | 同ポップアップでタスクの期間を日数（数値入力）で指定できる | 必須 | マイルストーンの場合は期間入力欄を表示せず開始日のみ編集すること |
| R-G14-03 | 入力値が不正（日付フォーマット違反・期間が1未満）の場合はエラーメッセージをポップアップ内に表示し適用をブロックする | 必須 | 正しい値が入力されるまでMermaidコードを変更しないこと |
| R-G14-04 | 依存関係（`after <id>`）を持つタスクで開始日を直接入力した場合、依存関係を解除して絶対日付に変換する | 必須 | 適用後にMermaidコードが `after <id>` から絶対日付形式に変わること（R-G11-05 と同方針） |
| R-G14-05 | `Enter` キーで適用、`Escape` キーでキャンセルできる | 必須 | ショートカットはポップアップ内のいずれの入力フィールドでも有効なこと |

## 8. データモデル

### 8.1 ガントチャート

```typescript
interface GanttData {
  title: string;
  dateFormat: string;        // 例: "YYYY-MM-DD"
  axisFormat?: string;       // 例: "%m/%d"
  sections: GanttSection[];
}

interface GanttSection {
  name: string;              // セクション名（セクションなしの場合は空文字）
  tasks: GanttTask[];
}

interface GanttTask {
  id: string;                // タスクID（省略時は自動付番）
  label: string;             // 表示ラベル
  status: 'done' | 'active' | 'crit' | '';
  startDate: string;         // 絶対日付（dateFormat準拠）
  duration: number;          // 日数
}
```
