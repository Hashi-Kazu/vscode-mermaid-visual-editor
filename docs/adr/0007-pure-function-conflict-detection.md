# ADR 0007: コンフリクト検知ロジックを VS Code 非依存の純粋関数として分離

**日付**: 2026-06-17（v2.3.0、commit 8b459cc）  
**ステータス**: 採用済み  
**確信度**: 高

---

## コンテキスト

ADR 0006 で導入したコンフリクト検知ロジック（base スナップショットとの比較）を実装する際、ロジックの配置場所を決定する必要があった。`EditorPanel` クラスのメソッドとして実装する方法と、独立したモジュールとして分離する方法がある。

## 決定

コンフリクト判定ロジック（`detectConflict`）と改行正規化（`normalizeText`）を **VS Code API に依存しない純粋関数** として `src/conflictDetection.ts` に分離する。

## 理由

コミットメッセージ（8b459cc）に明記: "判定ロジックは VS Code 非依存の純粋関数として分離し test/conflictDetection.test.ts（9ケース）で検証"。

VS Code の Extension Host で実行するコードは `vscode` モジュールに依存するため、Node.js の通常テストランナーで直接テストできない。純粋関数として分離することで、`vscode` モジュールのモック不要でユニットテストを記述できる。

また、コンフリクト検知は副作用のない純粋なロジック（base・current・outgoing の3テキスト比較）であるため、関数分離が自然な設計。

## 捨てた選択肢

- **EditorPanel のプライベートメソッドとして実装**: テストするには VS Code API をモックする必要があり、テストの複雑度が増す。ロジックがクラス実装に埋もれて可読性も低下する。

## 影響

- `test/conflictDetection.test.ts` で 9 ケースをテスト（`node --test` で実行）。
- 同パターンを踏まえて、テスト対象の純粋ロジックは `src/` に置く方針（`refactor: テスト容易化のため純粋ロジックを抽出` v2.6.2 も同様）。
